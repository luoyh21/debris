数据来源与覆盖范围

系统整合了六大权威空间态势感知数据源，通过统一的 NORAD 编号去重机制，构建了覆盖 60,000+ 在轨目标的综合目录；此外还独立接入 Asterank 小行星 / 近地天体目录，用于太阳系尺度的专题分析。每个数据源提供不同维度的信息，相互补充，确保数据的完整性和可靠性。

各数据源详解

Space-Track.org

由美国太空监视网络 (SSN) 维护，是系统的核心轨道数据源。提供每个在轨目标的两行轨道根数 (TLE)、基本目录信息（名称、NORAD ID、国家代码、目标类型、发射日期）以及空间轨迹段数据。TLE 数据用于 SGP4 轨道传播预报。

UCS 卫星数据库

忧思科学家联盟 (Union of Concerned Scientists) 维护的活跃卫星数据库。为载荷目标补充卫星用途（通信、遥感、导航、科学研究等）和用户类型（军用、民用、商用、政府）信息，帮助分析空间资产的功能分布。

ESA DISCOS

欧空局碎片监测数据库。提供物理参数如质量和截面积、任务名称、以及在轨/已衰减状态标识。这些数据对于碰撞风险评估中的联合硬体半径 (HBR) 计算尤为重要。

GCAT

天文学家 Jonathan McDowell 维护的全球航天活动目录，提供从 1957 年至今的完整发射记录，包括国别统计和年度趋势。用于系统概览面板中的发射历史趋势图表。

UNOOSA

联合国外层空间事务厅维护的官方发射登记数据。作为交叉验证数据源，与 GCAT 数据进行对比，确保发射统计的准确性。

Asterank

独立的小行星 / 近地天体（NEO）专题库，基于 JPL Small-Body Database 扩展了经济价值估算字段（price、profit、Δv）。在系统中与地球在轨目标目录分离存储于 external_asterank 表，可在 目标目录的「小行星 / NEO (Asterank)」标签页筛选查看。由 scripts/ingest_asterank.py 导入。

数据时效与新鲜度

💡 提示：在系统概览面板的「多源数据摄入状态」区域，您可以看到每个数据源的实际入库记录数量和最后更新时间，确认数据是否处于最新状态。

数据更新说明

Space-Track TLE 是最频繁更新的数据源，大多数目标的 TLE 每 8–24 小时刷新一次。轨道传播预报的精度与 TLE 的时效性直接相关——TLE 越新鲜，SGP4 传播误差越小。

UCS 和 UNOOSA 更新频率较低，但这些数据源提供的用途分类和宏观统计信息变化缓慢，不影响日常使用。

ESA DISCOS 的物理参数（质量、截面积）在目标入轨后基本不再变化，因此更新频率无需很高。
