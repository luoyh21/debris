规避策略生成器

场景 在碰撞风险评估检出高风险合取事件后，自动给出可执行的规避机动方案：从化学脉冲、电推持续燃烧到上升段时空走廊三类推进/相位场景。

⚡ 前置条件：需要先运行 轨迹仿真 与 碰撞风险，页面自动从 session 读取最严重事件；也可手动输入 TCA 与最近距离。

方案推进类型核心算法典型 ΔV响应时间适用场景

① 脉冲规避（B 平面解析法）

B 平面是与相对速度 v_rel 垂直的平面，在该平面上的位置矢量即「错失距离矢量」。

正交基（Öpik 约定）：

η̂  = v_rel / |v_rel|             (B 平面法向)
ζ̂  = (v_rel × h_pri) / |·|       (B 平面 down-track)
ξ̂  = ζ̂ × η̂                       (B 平面 cross-track)

HCW 状态转移矩阵给出 ΔV → TCA 时刻位置偏移 的线性敏感度：

Δr_TCA(ECI) ≈  R_LVLH→ECI · Φ_rv(Δt) · R_LVLH→ECIᵀ · Δv_ECI

其中 Δt = TCA − t_man

把 3D 位置偏移投影到 B 平面 (ξ̂, ζ̂)：(Δξ, Δζ) = P · J · Δv， 则在 ‖Δv‖ ≤ Δv_max 约束下：

max  ‖P J Δv‖²   s.t.  ‖Δv‖ ≤ Δv_max
↓ Lagrange 乘子
Δv* = Δv_max · v₁(SVD(M))      M = P · J

主右奇异向量 v₁ 即最优方向；符号选择保证 错失距离朝增大方向。 推进剂消耗按 Tsiolkovsky 反推：m_p = m₀(exp(|Δv|/v_e) − 1)，v_e = Isp · g₀。

💡 解析法的优势：毫秒级返回首选机动方案，无须迭代，特别适合在防撞预警生成后的应急响应窗口（数十秒至数分钟）。受限于 J2 等扰动忽略，建议结合高精度数值外推做二次校核。

② 持续小推力（SCP / SOCP-lite）

电推系统推力幅值 ≪ 化学推进，但作用时长远超脉冲。把推力建模为 LVLH 系常值加速度 α，敏感度变为 Φ_rv 在燃烧窗口的积分：

B(T)  =  ∫₀ᵀ Φ_rv(T − τ) dτ          (32 点 Gauss 数值积分)
Δr_TCA  ≈  R · B(T) · Rᵀ · α
‖α‖    ≤  T_thrust / m_avg   (推力上限)

线性化后凸优化目标完全等价于 B 平面情形：

max ‖P · B · α‖²    s.t.  ‖α‖ ≤ T_thrust / m

一次迭代即收敛于线性解（SOCP-lite）；对于多目标 / 多碎片场景，把不同事件的 P_i · B_i 堆叠为多约束二阶锥规划即为完整 SCP，通常 3 – 5 次迭代收敛。

③ 上升段时空走廊 + MPC

火箭上升段必须遵守气动载荷（Q-α 包络）、结构强度、重力转向轨迹等严约束。可用控制只剩两个连续量：

  • Δaz：发射方位微调
  • Δθ：俯仰程序偏置

它们对 TCA 时刻位置的一阶敏感度为：

∂(lateral) / ∂Δaz   ≈   |v| · (T_TCA − t_man) · π/180
∂(radial)  / ∂Δθ    ≈   ½ |v| · (T_TCA − t_man) · π/180

把碎片云膨胀边界投影成时变时空矩形，构造一维线性规划：

max miss   s.t.   |Δaz| ≤ Δaz_max,  |Δθ| ≤ Δθ_max

实时引导可在地面遥测 1 Hz 周期上滚动重解，构成 MPC 闭环；本工程实现给出第一迭代解析解，便于离线策略评估与可视化。

页面交互流程

  1. 页面打开后自动从 session 读取「碰撞风险」最严重事件，或允许手动输入 TCA、最近距离、|v_rel|；
  2. 选择σ（位置不确定度）与 HBR（联合硬体半径）；
  3. 三个 Tab 各自有专属参数（ΔV 预算、Isp、点火提前量、推力、燃烧时长、走廊宽度等）；
  4. 每个 Tab 自动给出：ΔV 矢量、修正后错失距离与 Pc、推进剂消耗、3D 轨迹对比图（原 vs 规避后）；
  5. 页面末尾「三种规避方案对比」表横向展示三类机动指标，方便决策。

输出指标

指标说明
错失距离前 / 后当前与机动后预测的 TCA 错失距离 (m)

调用方式（Python）

from avoidance import (
    inputs_from_event,
    optimal_impulsive_dv,
    design_low_thrust_burn,
    design_ascent_correction,
)

inp = inputs_from_event(event, sim_result, sigma_combined_km=1.5, hbr_km=0.02)

sol1 = optimal_impulsive_dv(inp,  dv_budget_kms=0.005, maneuver_lead_s=900)
sol2 = design_low_thrust_burn(inp, burn_duration_s=600, thrust_N=1.0,
                                spacecraft_mass_kg=4500, isp_s=3000)
sol3 = design_ascent_correction(inp, t_launch_s=0.0,
                                  max_dazimuth_deg=3.0, max_dpitch_deg=1.0)

print(sol1.dv_mag_kms, sol1.miss_after_km, sol1.pc_after, sol1.propellant_kg)
