Ansys STK 跨算法验证

场景 把本系统轨道传播 / 6-DOF 仿真结果与 Ansys STK（Systems Tool Kit，业界 SSA 行业标准）做位置 / 速度 RMS 和 RIC（Radial / In-track / Cross-track）误差对照，自动给出 通过 / 偏差超阈值 结论并写回本页面。

验证策略：
  1. SGP4 → STK SGP4 —— 给定 TLE，把卫星放进 STK 并使用其 SGP4 推演；与本系统 propagator/sgp4_propagator.py 输出对照位置 / 速度 RMS。 Vallado 算法两端理论上完全一致，是实现层面的回归基准。
  2. 6-DOF → STK HPOP —— 取仿真末端 ECI 状态作为初值，用 STK HPOP（EGM2008 + NRLMSISE-00 + 月日扰动） 作为「地面真相」，向前推 N 分钟，与本系统等价数值积分对照。
  3. RIC 分解 —— 不仅看总位置 RMS，还把误差投影到 Radial / In-track / Cross-track 三个方向。沿轨向（In-track）误差占主导是 SGP4 / 简化大气模型的典型指纹，可直接指导算法迭代方向。
跨平台说明：Ansys STK Engine 仅 Windows 原生支持（PySTK / COM）。
  • 当 主机为 Windows + 已安装 STK 12.1+：系统会调用 STK 真值，并把结果写入 data/validation/stk_validation.json，本文档下方实时读取。
  • 当 主机为非 Windows 或 未安装 STK：UI 上"运行 STK 验证"按钮自动禁用 / 提示， 但仍可使用参考实现真值（独立 sgp4 库 + HPOP-lite RK45 积分）做自洽性 回归，以保证算法可证伪。

3.1 算法升级：三变体 ↔ STK HPOP

本系统按本文档第四节「RIC 误差诊断」的建议修正，提供三档力学模型：

  • baseline —— 与 trajectory/six_dof.py 一致：J2 + USSA-76 大气；
  • optimized —— J2 + J3 + J4 zonal 高阶 + 月日点质量扌动 + 太阳辐射压（球面）+ USSA-76；
  • EGM 4×4 —— 用 EGM96 球谐 4×4（含 12 项 sectorial / tesseral）替换 J2 引力，并保留月日 + SRP + USSA-76。 球谐势函数在 ECEF 中用 Holmes-Featherstone 归一化 ALF 递推算出，再做三阶中心差分得到加速度； ECI↔ECEF 用 GMST 单旋（IAU 1982），实现见 stk_validation/gravity_egm.py。
  • EGM 6 (2026-05 新增) —— EGM96 球谐 6×6（共 33 项 C̄/S̄）+ 月日 + SRP + USSA-76。 在 4×4 之上把 zonal/sectorial/tesseral 阶数全部抬到 6 阶， LEO 6h 长期 In-track 误差再压 75%。当前推荐档。
  • EGM 8 + MSISE (2026-05 新增) —— EGM96 球谐 8×8（共 60 项 C̄/S̄）+ 月日 + SRP + NRLMSISE-00 大气（Picone 2002，含太阳活动 / 地磁；缺库自动回退 USSA-76）。 短弧（≤ 30 min）下通常优于 EGM6；长弧表现取决于 F107/Ap 是否与 STK 内部 SpaceWeather.spw 匹配。

所有变体使用同一份 STK HPOP 真值，所以 RMS 差额完全反映 candidate 物理升级的效果。 最终对照（ISS-like LEO 408 km，inc 51.6°）：

30 min（避撞窗口） 289 m276 m (−5%)115 m (−60%) 52 m (−82%)35 m (−88%)EGM 8+MSISE
6 h（长期 ≈ 4 圈） 4 005 m3 957 m (−1%)785 m (−80%) 199 m (−95%)437 m (−89%)EGM 6

括号内 = 相对 baseline 的改善。EGM 8+MSISE 在 30 min 短弧下因加入大气模型而获得最佳成绩； 6 h 长弧由于 F107=165 sfu 比 STK HPOP 默认偏高，NRLMSISE 给出的密度偏大，反而引入额外 In-track 偏差，因此 长期推荐档 = EGM 6 + USSA-76。完整自动消融脚本见 scripts/stk_hpop_ablation.py，单次端到端再验证脚本见 scripts/stk_run_hpop_validation.py。

3.2 EGM 4×4 / 6×6 / 8×8 实现简介 （stk_validation/gravity_egm.py）

球谐展开式（标准化形式）：

R(r) = (μ/r) · Σₙ₌₂ᴺ Σₘ₌₀ⁿ (Re/r)ⁿ · P̄ₙₘ(sinφ) · ( C̄ₙₘ cos(mλ) + S̄ₙₘ sin(mλ) )

其中：

  • 系数表：来自 ICGEM 的 egm96.gfc，硬编码在 _EGM96_C / _EGM96_S 字典中。 本次升级把系数表由 4×4 (12 项) 扩到 8×8 (60 项)，含完整 zonal / sectorial / tesseral： EGM96_MAX_DEGREE = 8。
  • 归一化 ALF P̄ₙₘ：用 Holmes-Featherstone (2002) 三段递推（对角 / 次对角 / 一般），数值稳定，无极点奇异。
  • 加速度：用 1 m 三阶中心差分 ∂R/∂(x,y,z)，再加中央项 −μr/r³，比解析归一化导数实现简单且数值精度足够（vs STK HPOP 误差 ≤ 200 m / 6h@8×8）。
  • 坐标：球谐势在 ECEF 中算，结果旋转回 ECI；ECI↔ECEF 用 GMST IAU 1982 单旋（忽略章动 / 极移），对 LEO 6h 位置贡献 < 50 m。
  • 大气：runner 同时支持 use_nrlmsise=True，调用 stk_validation/atmosphere.py 封装的 NRLMSISE-00（缺库自动回退 USSA-76）；ECI → ECEF → 几何经纬度 → density 调用按步执行。
✅ EGM 6 / EGM 8+MSISE 的实测改善（vs baseline）：
  • 30 min：289 m → 52 m (EGM 6) / 35 m (EGM 8+MSISE)（−82% / −88%）
  • 6 h：4 005 m → 199 m (EGM 6) / 437 m (EGM 8+MSISE)（−95% / −89%）
  • EGM 6 长期推荐：6 h In-track 主导项由 3.99 km → 144 m，Radial 由 269 m → 31 m。
说明： EGM 8 + NRLMSISE-00 在长弧反而比 EGM 6 略差， 原因是 F107 = 165 sfu 默认值比 STK HPOP 内部 SpaceWeather.spw 偏高，给出更高的大气密度 → candidate 阻力偏强 → In-track 偏差累积。在面板 / API 中可调 F107 / Ap 与 STK 一致后会进一步收敛。

四、RIC 误差诊断 — 错在哪里？

仅看总位置 RMS 不足以指导算法迭代。RIC 三向分解能告诉你误差类型：

主导方向 典型成因 建议修正
In-track（沿轨） 大气阻力 / 弹道系数估计偏差；TLE 中 BSTAR 与真实 (CD·A/m) 不匹配； 简化大气模型（USSA-76 vs NRLMSISE-00）。 升级到 NRLMSISE-00 / Jacchia-Bowman；引入弹道系数实时估计； 缩短 TLE 更新周期（< 24h）。
Cross-track（轨道法向） 升交点漂移；J2 / J3 / J4 引力位的展开阶数不足；地球岁差章动未引入。 把引力场从 J2 提升到 EGM96 / EGM2008，至少展开到 21×21； 引入 IAU 2000A 岁差章动模型。
Radial（径向） 初始状态向量本身有 1 σ 噪声；SGP4 在远日点附近的非线性误差； SRP（太阳辐射压）建模缺失。 引入定轨 / OD 流程对初值做协方差估计；启用 SRP 球面模型； 在远日点段加密积分步长。
📐 误差累积规律：
  • 沿轨误差线性累积 → 主要由阻力 / Δv 偏差驱动；调阻力模型即可。
  • 沿轨误差二次方 / 立方累积 → 弹道系数估计有系统偏差；建议引入实时 OD。
  • 径向 / 法向均小但沿轨大 → 模型几何正确，仅"大小"错；多半是单位 / 时基 (TT vs UTC) 不一致。

print(detect_stk_availability().to_dict())"

# 2) SGP4 vs STK / 参考真值（自动回退）
python -c "from stk_validation import run_sgp4_validation;\
r = run_sgp4_validation(line1='1 25544U ...', line2='2 25544 ...', \
                       norad_id=25544, duration_s=86400, step_s=600);\
print(r.short_summary())"

# 3) 6-DOF vs STK HPOP / HPOP-lite

import streamlit_app  # 拿 sim_result 来源\

"
