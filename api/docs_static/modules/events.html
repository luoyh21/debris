太空事件管理

场景 把解体、碰撞、再入、机动、合取预警 (CDM) 等在轨事件作为一等公民管理：自动从外部权威源拉取，本地手填补充，应用 NASA Standard Breakup Model 预测碎片产物，并按 CCSDS NDM 标准与外部 SSA 平台无损互通。

⚡ 本系统将事件视为预测未来风险的输入参数，而不仅是历史记录。任意一次解体 / 碰撞事件都可以「点击 → 一键 SBM 模拟 → 生成碎片云 → 直接进入避撞筛查」。

NASA Standard Breakup Model（解体建模核心）

当事件进入数据库后，系统并不只把它当成一个静态点——而是用 NASA Standard Breakup Model (Johnson et al. 2001) 即时生成数百到数千个碎片，每个碎片携带：

  • 特征长度 Lc (m) — 累积幂律分布
  • 面质比 A/M (m²/kg) — 对数高斯分布
  • 弹出速度 Δv 矢量 — log10|Δv| ~ N(0.9·log10(A/M)+μ, 0.4²)，方向各向同性

Lc 累积分布（Johnson 2001）

解体（爆炸）  :  N(>Lc) = 6 · Lc⁻¹·⁶                    (Lc 单位 m)
碰撞         :  N(>Lc) = 0.1 · M^0.75 · Lc⁻¹·⁷¹       (M = 联合质量 kg)

能量 / 质量比 ⇒ 灾难性判别

定义 E/M = ½ · μ · v_rel² / m_total（J/g）。 当 E/M ≥ 40 J/g 时事件被定义为灾难性，母体被认为完全粉碎，质量贡献按总质量计；否则只统计撞击体破碎贡献（NASA EVOLVE 4.0 约定）。

输出指标

指标说明
n_total抽样到的碎片总数（≤ max_fragments）
n_tracked_ge_10cm≥ 10 cm 可被 USSPACECOM SSN 编目的碎片数
n_lethal_ge_1cm≥ 1 cm 致命碎片数（NASA-STD-8719.14B 击穿威胁阈）
catastrophic是否完全粉碎

「事件管理 → NASA SBM 模拟」标签页中：选定一条解体或碰撞事件 → 选 Lc 范围与抽样上限 → 一键运行 → 立即给出 4 个指标卡 + 3D 碎片云（按尺寸/可追踪性着色）+ Lc 直方图。导出按钮一键打包为 OCM 文件，所有碎片状态作为 USER_DEFINED_FRAGMENT_xxxx 字段嵌入。

CCSDS NDM 互操作性

系统严格遵循 CCSDS Navigation Data Messages 标准。所有写入器输出 KVN（key-value-newline） 文本，是 Space-Track / SDC 默认分发格式。

消息类型标准代号用途本系统行为
CDM CCSDS 508.0-B-1 合取预警 导入 / 导出 — 含 TCA、MISS_DISTANCE、Pc、Foster-1992 标识
OPM CCSDS 502.0-B-3 单点轨道 + 机动 导出避撞机动方案（位置 + 速度 + MAN_DV）
OEM CCSDS 502.0-B-3 完整星历 已在「目标目录 OEM 导出」与「轨迹仿真 OEM 导出」实现，TEME 参考系，STK / GMAT 兼容
OCM CCSDS 502.0-B-3 综合任务计划 导出包含解体碎片清单的 OCM（USER_DEFINED_FRAGMENT_*）
RDM CCSDS 508.1-B-1 再入消息 导入 / 导出 — REENTRY_EPOCH + REENTRY_ALTITUDE

REST API 接口

方法路径说明
GET/api/v1/events列出事件，可按类型 / 来源筛选
GET/api/v1/events/{id}事件详情（含原始 raw 字典）
POST/api/v1/events插入事件（手填 / 第三方系统）
DELETE/api/v1/events/{id}删除事件
POST/api/v1/events/{id}/breakup-simulateNASA SBM 解体模拟，返回 ≤500 条碎片状态
GET/api/v1/events/{id}/export?format=cdm|opm|ocm|rdm导出 CCSDS NDM 文件
POST/api/v1/events/import导入 CCSDS KVN 文本（自动识别格式）

典型工作流（curl）

# 1. 创建一次碰撞事件
curl -X POST http://localhost:8502/api/v1/events \
  -H 'Content-Type: application/json' \
  -d '{"event_type":"COLLISION","epoch":"2026-04-15T08:30:00Z",
        "name":"DEMO-COLL","altitude_km":800,"inclination_deg":53,
        "mass_parent_kg":1500,"mass_target_kg":50,"energy_to_mass":85}'
# → {"id":1,"ok":true}

# 2. 跑 NASA SBM
curl -X POST 'http://localhost:8502/api/v1/events/1/breakup-simulate?lc_min_m=0.01&lc_max_m=1&max_fragments=2000'
# → {"n_total":2000, "n_tracked_ge_10cm":118, "catastrophic":true, ...}

# 3. 导出 CCSDS CDM
curl 'http://localhost:8502/api/v1/events/1/export?format=cdm' > event_1.cdm

# 4. 把 CDM 导回（往返测试）
curl -X POST http://localhost:8502/api/v1/events/import \
  -H 'Content-Type: application/json' \
  -d "{\"text\": \"$(cat event_1.cdm | jq -Rs .)\"}"
