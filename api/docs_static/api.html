API 使用指南

系统提供 19 个 REST API 接口，支持通过 HTTP 请求以编程方式调用核心分析能力——核心查询 / 仿真 6 类、数据源管理 6 类、太空事件管理 7 类。适用于外部系统集成、自动化脚本或批量数据获取。

通用响应格式

所有接口返回 JSON 格式的响应，包含以下通用字段模式：

{
  "count": 42,              // 匹配的总数量
  "showing": 42,            // 本次返回的数量（受 limit 限制）
  "objects": [...],         // 具体对象列表
  "summary": "...",         // 人类可读的摘要（中文）
  "method": "PostGIS"       // 使用的查询方法（仅部分接口）
}

错误处理

当请求出错时，响应中会包含 error 字段：

{
  "error": "错误描述信息"
}

快速示例

以下是一个完整的 curl 示例，查询文昌发射场附近的碎片：

curl -X POST http://<your-host>:8000/api/v1/debris/region \
  -H "Content-Type: application/json" \
  -d '{
    "lat_deg": 19.61,
    "lon_deg": 110.95,
    "radius_km": 500,
    "alt_min_km": 200,
    "alt_max_km": 2000,
    "object_type": "ALL",
    "limit": 20
  }'
💡 提示：将示例中的 <your-host> 替换为系统实际部署的主机名或 IP 地址。如果在本机访问，使用 localhost。

数据源管理（运行时热更）

本组接口面向运维 / 二次开发场景，提供运行时数据源优先级调整与自定义数据源 CRUD。系统统一目录 v_unified_objects 在每次写入或优先级变更后会自动刷新——当多个源对 同一 NORAD 编号都有数据时，优先级数字最小（默认 Space-Track = 10）的源胜出。

✅ 支持给「当前没有任何数据」的数据源预置优先级（用作占位），等之后真正接入时立刻按既定排序参与去重。

列出全部数据源

GET /api/v1/datasources

返回所有内置 / 用户自建数据源的优先级、描述、自建源行数：

curl http://localhost:8502/api/v1/datasources

{
  "sources": [
    {"source":"Space-Track","priority":10,"is_user_defined":false,"user_row_count":0, ...},
    {"source":"UCS",         "priority":20, ...},
    {"source":"ESA-DISCOS",  "priority":30, ...},
    ...
  ]
}

修改 / 创建优先级

PUT /api/v1/datasources/{source}/priority 
# 把 Space-Track 优先级降到 25（让自定义源更高）
curl -X PUT http://localhost:8502/api/v1/datasources/Space-Track/priority \
     -H 'Content-Type: application/json' \
     -d '{"priority":25}'

# 创建尚无数据的占位源
curl -X PUT http://localhost:8502/api/v1/datasources/futureSrc/priority \
     -H 'Content-Type: application/json' \
     -d '{"priority":3,"description":"未来高优先级源"}'

响应：{"source":"futureSrc","priority":3,"created":true}

写入 / 修改单行（与 Space-Track 同字段）

POST /api/v1/datasources/{source}/objects

请求体字段与 catalog_objects（Space-Track 主表）完全一致；按 (source, norad_cat_id) 幂等 upsert。如果 source 不在 datasource_priority 中会自动以默认优先级 100 注册。

curl -X POST http://localhost:8502/api/v1/datasources/myCustomSrc/objects \
     -H 'Content-Type: application/json' \
     -d '{
        "norad_cat_id": 990001,
        "name": "TEST-SAT-A",
        "object_type": "PAYLOAD",
        "country_code": "PRC",
        "launch_date": "2026-04-01",
        "inclination": 53.0,
        "perigee_km": 540, "apogee_km": 560,
        "period_min": 95.5,
        "rcs_size": "MEDIUM",
        "extra": {"campaign":"test","note":"…"}
     }'

批量写入

POST /api/v1/datasources/{source}/objects/bulk

请求体为对象数组（最多 500 个），效果等同多次单写。

删除

方法路径语义
DELETE/api/v1/datasources/{source}/objects/{norad_cat_id}删除某行
DELETE/api/v1/datasources/{source}删除整个自建源（含全部行）

受保护：试图删除 Space-Track / UCS / ESA-DISCOS / GCAT / UNOOSA / Asterank / TechPort 等内置源会返回 HTTP 403。

统一视图与自动刷新

每次以上接口写入或更新后，FastAPI 端会自动 REFRESH MATERIALIZED VIEW v_unified_objects。 目录页、可视化探索、统计等所有依赖该视图的页面会立刻看到新去重结果。

关于 Asterank 与 NASA TechPort 专题数据

除上述数据源管理 6 个接口外，前 6 个 REST 接口全部作用于地球在轨目标（Space-Track 主目录 + UCS/ESA 增强字段），不涉及 Asterank 小行星 / 近地天体（NEO）专题库与 NASA TechPort 航天技术项目组合。要访问这两个专题数据，可通过以下方式：

  • 交互式浏览与筛选：前往 目标目录 页面的「小行星 / NEO (Asterank)」与「NASA 技术项目 (TechPort)」标签页，按各自字段筛选。
  • 检查入库条数：调用 /api/v1/stats，其 external_sources.asterank 与 external_sources.techport 字段返回 external_asterank / external_techport 表中的记录数。
  • 直接 SQL 查询：用任何 PostgreSQL 客户端连接数据库，查询 external_asterank 或 external_techport 表；字段含义见 目标目录 页面对应标签的数据列说明。

NASA TechPort 公开端点无需 token；若遇到限流或需要私有 / 草稿数据，可在 techport.nasa.gov/help/api 复制当前会话 token 并填入 .env 的 NASA_TECHPORT_TOKEN=，每次刷新 TechPort 站点都会换 token，请按需更新。
