API 使用指南

系统提供 6 个 REST API 接口，支持通过 HTTP 请求以编程方式调用核心分析能力。适用于外部系统集成、自动化脚本或批量数据获取。

基本信息

属性说明
基础 URLhttp://<your-host>:8000
认证方式无需认证，开放访问
数据格式请求体：JSON（POST 请求）
响应体：JSON
编码UTF-8
Swagger 文档打开 Swagger UI（交互式 API 文档）
ReDoc 文档打开 ReDoc（只读 API 文档）
💡 Swagger UI 提供交互式的 API 测试界面，您可以直接在浏览器中填写参数、发送请求并查看响应，无需任何编程。

通用响应格式

所有接口返回 JSON 格式的响应，包含以下通用字段模式：

{
  "count": 42,              // 匹配的总数量
  "showing": 42,            // 本次返回的数量（受 limit 限制）
  "objects": [...],         // 具体对象列表
  "summary": "...",         // 人类可读的摘要（中文）
  "method": "PostGIS"       // 使用的查询方法（仅部分接口）
}

错误处理

当请求出错时，响应中会包含 error 字段：

{
  "error": "错误描述信息"
}

快速示例

以下是一个完整的 curl 示例，查询文昌发射场附近的碎片：

curl -X POST http://<your-host>:8000/api/v1/debris/region \
  -H "Content-Type: application/json" \
  -d '{
    "lat_deg": 19.61,
    "lon_deg": 110.95,
    "radius_km": 500,
    "alt_min_km": 200,
    "alt_max_km": 2000,
    "object_type": "ALL",
    "limit": 20
  }'
💡 提示：将示例中的 <your-host> 替换为系统实际部署的主机名或 IP 地址。如果在本机访问，使用 localhost。
