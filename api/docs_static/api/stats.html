系统概况统计

GET /api/v1/stats

适用场景 快速获取系统数据库的概况信息——目标总数、各类型分布、GP 元素和轨迹段的入库数量，以及多源融合视图 v_unified_objects 和各个外部 / 专题数据源（GCAT / UNOOSA / UCS / ESA DISCOS / Asterank）的入库条数。适用于健康检查、监控面板集成或数据摄入状态确认。

请求参数

此接口无需任何参数，直接发送 GET 请求即可。

响应字段

字段类型说明
catalog_totalint目标目录 (catalog_objects) 中的总记录数（Space-Track 主目录）
by_typeobject按目标类型分组的数量。键为类型名称，值为数量
by_type.DEBRISint碎片数量
by_type.PAYLOADint有效载荷数量
by_type["ROCKET BODY"]int火箭箭体数量
by_type.UNKNOWNint未知类型数量
gp_elements_countintGP 轨道根数 (gp_elements) 表的总记录数
trajectory_segments_countint轨迹段 (trajectory_segments) 表的总记录数
unified_totalint多源融合视图 v_unified_objects 去重后的空间目标总数（融合 Space-Track + UCS + ESA DISCOS）
unified_primary_sourcesint多源融合视图中的 primary_source 种类数（正常为 3：Space-Track / UCS / ESA-DISCOS）
external_sourcesobject各外部 / 专题数据源表的入库条数（见下表）
external_sources.ucsintUCS 卫星数据库 (external_ucs_satellites) 条数
external_sources.esa_discosintESA DISCOS (external_esa_discos) 条数
external_sources.gcat_yearly_launchesintGCAT 按年/国别/类型的发射统计 (external_yearly_launches)
external_sources.gcat_cumulative_onorbitintGCAT 逐年累计在轨统计 (external_cumulative_onorbit)
external_sources.gcat_country_yearly_payloadintGCAT 按年/国别的有效载荷发射量 (external_country_yearly_payload)
external_sources.gcat_onorbit_snapshotintGCAT 当前在轨快照 (external_onorbit_snapshot)
external_sources.unoosaintUNOOSA 年度发射统计 (external_unoosa_launches)
external_sources.asterankintAsterank 小行星 / NEO 专题库 (external_asterank) — 与地球在轨目标独立

结果解读

📊 关键指标含义

指标正常范围异常说明
catalog_total40,000–70,000如果为 0，说明尚未完成 Space-Track 数据摄入
gp_elements_count应接近 catalog_total如果远小于 catalog_total，说明 GP 数据摄入不完整
trajectory_segments_count> 0如果为 0，区域碎片查询将退化为慢速的 SGP4 模式
by_type.DEBRIS通常占总数 > 40%碎片是最大的目标类别
unified_total应 ≥ catalog_total若为 0，通常是 v_unified_objects 物化视图未创建，运行 python run.py refresh-views 即可
external_sources.asterank数千条（随 --limit 而定）若为 0 说明尚未运行 python scripts/ingest_asterank.py
💡 提示：此接口可用于自动化监控脚本。例如定期检查 catalog_total 是否增长，确认数据摄入流程正常运行。
