按 RCS 大小筛选碎片

POST /api/v1/debris/rcs

适用场景 按雷达截面积 (RCS) 大小类别筛选空间目标。适用于威胁甄别——例如仅筛选 LARGE 目标评估对载人航天器的威胁，或筛选 SMALL 目标了解不可追踪碎片的目录覆盖情况。

RCS 类别说明

类别RCS 范围典型目标威胁特征
SMALL< 0.1 m²小碎片、螺栓、碎片云颗粒追踪困难，距离不确定性最大
MEDIUM0.1 – 1 m²中等碎片、小型卫星残骸可追踪但位置精度一般
LARGE> 1 m²大型碎片、箭体、失效卫星最易追踪，碰撞后碎片产生最多

请求参数

参数名类型必填默认值说明
rcs_sizesarray[string]否全部类别RCS 类别列表：["SMALL"] / ["MEDIUM","LARGE"] 等
alt_min_kmfloat否0最低近地点高度 (km)
alt_max_kmfloat否2000最高远地点高度 (km)
object_typestring否"ALL"目标类型：ALL / DEBRIS / PAYLOAD / ROCKET BODY
limitint否50最大返回条数，1–200

响应字段

字段类型说明
filterobject筛选参数回显
countint匹配总数量
showingint本次返回数量
objectsarray目标列表
objects[].norad_cat_idintNORAD 编号
objects[].namestring目标名称
objects[].object_typestring目标类型
objects[].countrystring国家代码
objects[].rcs_sizestringRCS 类别：SMALL / MEDIUM / LARGE
objects[].perigee_kmfloat近地点高度 (km)
objects[].apogee_kmfloat远地点高度 (km)
objects[].inclination_degfloat轨道倾角 (°)
objects[].period_minfloat轨道周期 (分钟)
summarystring中文摘要

curl 示例

# 筛选 LEO 中的 LARGE 碎片
curl -X POST http://<your-host>:8000/api/v1/debris/rcs \
  -H "Content-Type: application/json" \
  -d '{
    "rcs_sizes": ["LARGE"],
    "alt_min_km": 200,
    "alt_max_km": 2000,
    "object_type": "DEBRIS",
    "limit": 20
  }'

💡 使用场景示例：载人任务风险评估时，重点关注 LARGE 目标（碰撞后产生碎片最多）；目录完整性分析时，关注 SMALL 目标（追踪覆盖率最低的类别）。
